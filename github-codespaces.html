üí∞ Pricing Structure

‚ö†Ô∏è Important: GitHub Codespaces is a paid service. Make sure you understand the pricing before use to avoid unexpected charges.

üí≥ Pay-as-You-Go Model

GitHub Codespaces uses a consumption-based pricing model:

Core-hours: $0.18 per core hour Storage: $0.07 per GB per month Bandwidth: $0.10 per GB Example: 2-core machine for 8 hours = 16 core-hours √ó $0.18 = $2.88

üéÅ Free Tier

GitHub provides monthly free allowances:

  • GitHub Free: 120 core-hours + 15GB storage
  • GitHub Pro: 180 core-hours + 20GB storage
  • GitHub Team: 240 core-hours + 25GB storage
  • GitHub Enterprise: 500+ core-hours

üí° Cost Optimization Tips

Reduce your Codespaces costs with these strategies:

  • Stop your codespace when not in use
  • Use smaller machine types for development
  • Delete old codespaces to save storage
  • Set up spending limits and alerts
  • Use Codespaces efficiently for complex tasks only

üìä Machine Types

Choose the right machine for your needs:

2-core: Basic web development 4-core: Full-stack development 8-core: Machine learning, complex builds 16-core: Heavy computational tasks Each core costs $0.18 per hour

üõ†Ô∏è Setting Up Your First Codespace

Prerequisites

Before starting, ensure you have:

  • Active GitHub account
  • Access to a repository
  • Billing information configured
  • Codespaces enabled (check Settings > Codespaces)

Create a Codespace

Navigate to your repository and click "Code" ‚Üí "Codespaces" ‚Üí "Create codespace on main"

Tip: You can also create a Codespace from any public repository on GitHub

Choose Machine Type

Select the appropriate machine for your needs:

  • 2-core: Basic web development (recommended)
  • 4-core: Full-stack development
  • 8-core: Complex applications, machine learning
  • 16-core: Heavy computational tasks

Wait for Environment Setup

Your Codespace will be created and configured automatically. This process typically takes 2-3 minutes and includes:

  • Container image download
  • Dependencies installation
  • VS Code server startup
  • Extension loading

Start Coding

Once ready, VS Code will open in your browser. You now have:

  • Full file system access
  • Integrated terminal
  • Git integration
  • Port forwarding for web servers
  • All installed tools and dependencies

üéØ Example: Setting Up a React App

Here's how to set up a React application in Codespaces:

// In your Codespace terminal npx create-react-app my-app cd my-app npm start // Your app will be accessible at: // https://your-codespace-xxxxx.devspaces.aws.amazon.com:3000

Codespaces automatically forwards port 3000, making your React app accessible immediately!
