Step 1: Create Your GitHub Account

Visit GitHub.com and Sign Up

Go to github.com and click the "Sign up" button. You'll need to provide:

  • A valid email address
  • A unique username (this will be part of your website URL)
  • A strong password
Pro Tip: Choose a username carefully! Your username determines your GitHub Pages URL (username.github.io).

Verify Your Email Address

Check your email for a verification message from GitHub. Click the verification link to activate your account. This step is crucial for enabling GitHub Pages access.

Complete Your Profile

Add a profile picture, bio, and other optional information. While not required, a complete profile helps with collaboration and makes your account look professional.

Enable Two-Factor Authentication (Recommended)

Go to Settings > Security and enable 2FA for enhanced security. This protects your repositories and enables access to advanced features.

Step 2: Create Your Repository

üè† For User/Organization Sites

Create a repository named exactly username.github.io where "username" is your GitHub username. This creates a special repository that becomes your main website.

Repository name: yourusername.github.io Visibility: Public (required for free hosting) Initialize: Add README file

üìÅ For Project Sites

Create any repository name (e.g., my-awesome-project) and GitHub Pages will serve it from a subdirectory at username.github.io/project-name.

Repository name: project-name Visibility: Public (for free) Deploy from: main branch

Create New Repository

Click the "+" icon in the top right corner and select "New repository". Fill in the repository name following the naming conventions above.

Set Repository Visibility

For free GitHub Pages hosting, your repository must be public. Private repositories require a paid GitHub plan.

Initialize Repository

Check "Add a README file" to create an initial commit. You can also add a .gitignore file and choose a license if needed.

Create Repository

Click "Create repository" to finish setup. You'll be redirected to your new repository page.

Step 3: Deploy Your Website

üéâ Quick Start: GitHub Pages supports automatic deployment from your main branch. Your site will be live within minutes!

Upload Your Website Files

You have several options to get your code into the repository:

  • Web Interface: Click "uploading an existing file" and drag & drop your HTML, CSS, JS files
  • GitHub Desktop: Clone the repository and use Git commands
  • Command Line: Use Git commands in terminal
  • GitHub Mobile: Use the mobile app for simple uploads

Commit Your Changes

After uploading files, scroll down and add a commit message describing what you've added. Then click "Commit changes".

Commit message: "Add initial website files" Description: "Upload homepage, stylesheet, and basic website structure"

Enable GitHub Pages

Go to the repository Settings tab, scroll down to "Pages" in the left sidebar, and configure your deployment:

  • Source: "Deploy from a branch"
  • Branch: "main" (or "master")
  • Folder: "/ (root)" for most projects
Important: Wait 5-10 minutes after enabling Pages for your site to become available.

Verify Your Site is Live

Your site will be available at:

User/Org sites: https://username.github.io Project sites: https://username.github.io/repository-name

Visit this URL to see your deployed website!

üì± Mobile Development with GitHub

GitHub Mobile App

Download the GitHub Mobile app for iOS and Android to manage your repositories on the go.

  • Upload files directly from your device
  • Edit Markdown files
  • Manage pull requests and issues
  • Review code changes
  • Access repositories offline

Mobile-Friendly Editors

Several mobile apps allow you to edit code on your phone or tablet:

  • GitJournal: Git client for Android
  • Working Copy: Full Git client for iOS
  • DroidScript: JavaScript development
  • JSRun: Browser-based code editor

Install GitHub Mobile

Download the GitHub Mobile app from App Store or Google Play. Sign in with your GitHub account credentials.

Navigate to Your Repository

Find your website repository in the app. You can search for it or find it in your repository list.

Upload or Edit Files

Use the "+" button to upload new files or edit existing ones. The mobile interface allows basic text editing and file uploads.

Commit and Deploy

After making changes, commit them with a message. GitHub will automatically redeploy your site using GitHub Pages.

üåê Custom Domain Configuration

Adding Your Own Domain

While GitHub Pages provides free hosting, you can also use your own custom domain (e.g., mywebsite.com) instead of the default username.github.io URL.

Purchase a Domain

Buy a domain from registrars like Namecheap, GoDaddy, or Google Domains. Popular options include:

  • Namecheap (.com, .org, .net)
  • Google Domains (.com, .net, .org)
  • Cloudflare (.com, .net, .org)
  • GitHub also offers .github.io domains

Configure DNS Records

In your domain registrar's control panel, add these DNS records:

Type: A Name: @ Value: 185.199.108.153 TTL: 3600 Type: A Name: @ Value: 185.199.109.153 TTL: 3600 Type: A Name: @ Value: 185.199.110.153 TTL: 3600 Type: A Name: @ Value: 185.199.111.153 TTL: 3600

Add CNAME Record (Optional)

For www subdomain support, add a CNAME record:

Type: CNAME Name: www Value: username.github.io TTL: 3600

Configure GitHub Pages

In your repository Settings > Pages, add your custom domain in the "Custom domain" field and save. GitHub will verify the domain and issue an SSL certificate automatically.

üöÄ Advanced Features & Best Practices

Build Automation

GitHub Actions can automatically build and deploy your site:

# .github/workflows/deploy.yml name: Deploy to GitHub Pages on: push: branches: [ main ] jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Deploy to GitHub Pages uses: peaceiris/actions-gh-pages@v3

Static Site Generators

Popular SSG options that work great with GitHub Pages:

  • Jekyll: Built into GitHub Pages
  • Next.js: Export static sites
  • Nuxt.js: Vue.js framework
  • VuePress: Vue.js documentation
  • Docusaurus: React documentation

Performance Optimization

Improve your site's performance on GitHub Pages:

  • Use HTTPS (automatic)
  • Optimize images (WebP format)
  • Minify CSS and JavaScript
  • Implement lazy loading
  • Use a CDN for large assets

SEO Best Practices

Optimize your GitHub Pages site for search engines:

  • Add meta descriptions
  • Use semantic HTML
  • Create sitemap.xml
  • Add robots.txt
  • Use proper heading structure
